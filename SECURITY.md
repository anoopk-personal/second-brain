# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly.

**Preferred method:** Use
[GitHub's private vulnerability reporting](https://github.com/anoopk-personal/second-brain/security/advisories/new)
to submit a report. This keeps the details confidential until a fix is available.

**Alternative:** Email anoopkumar at the address listed in the git commit history.

Please include:

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if you have one)

I will acknowledge reports within 72 hours and aim to release a fix within 7 days for critical issues.

## Scope

This policy covers:

- Documentation in this repository (setup guide, companion prompts)
- Edge function code published in the setup guide
- CI/CD configuration

Out of scope:

- Supabase platform vulnerabilities (report to [Supabase](https://supabase.com/security))
- OpenRouter platform vulnerabilities (report to [OpenRouter](https://openrouter.ai))
- Slack platform vulnerabilities (report to [Slack](https://slack.com/security))
